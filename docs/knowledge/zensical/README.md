# Zensical Knowledge Base for unitrie-rs

This folder is the local knowledge base for `unitrie-rs`.

## Purpose
1. Keep auditable implementation history.
2. Track parity/performance evidence.
3. Preserve architectural decisions for future integrations (including Reth-oriented reuse).

## Minimum update set per work batch
1. Update `worklog.md` and `worklog.json`.
2. Update `TODO-CODEX.md` statuses.
3. If process changed, update `AGENTS.md` and `maps/documentation-protocol.md`.

## Layout
- `evidence/project/worklog.md`: human-readable timeline.
- `evidence/project/worklog.json`: machine-readable timeline.
- `maps/documentation-protocol.md`: operational rules and checklist.
